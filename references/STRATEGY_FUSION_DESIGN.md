## 一、当前架构总览

### 1.1 整体管线

V3.0.0 采用**六池采样 + 跨期模式识别 + 策略融合引擎**三路并行的架构：

```
历史数据(N期)
    │
    ├──→ MultiPoolSampler (6池 × 前后区 = 12生成器)
    │     hot(30%) | cold(15%) | balance(20%)
    │     trend(20%) | game_theory(10%) | genetic(5%)
    │     → 每池生成 top-N 候选
    │
    ├──→ PatternRecognizer (第7池, 9种特征)
    │     span(12%) | consecutive(10%) | sum(12%) | parity(10%)
    │     repeat(15%) | tail(10%) | zone(12%) | prime(7%) | AC(12%)
    │     → 高频模式反推号码集合 + 评分增强 ×1.35
    │
    └──→ StrategyFusionEngine (5组多维度融合, 本文档核心)
          组1: 热号重号均衡 (加权投票)
          组2: 冷号遗漏反转 (贝叶斯融合)
          组3: 趋势动量 (博弈论均衡)
          组4: 博弈论冷门 (博弈论均衡)
          组5: ML概率融合 (ML概率融合)
          → 硬约束过滤 → 软约束评分 → 全局排序
    │
    ▼
融合层: 三路候选去重 + 综合评分 + 多样性boost + 近期加成
    │
    ▼
后处理: 隔期重号增强(SkipRepeatBooster) / 智能重号惩罚(SmartRepeatPenalty)
        → 尾号聚合检测 → AC值跟踪评分 → 偏差仪表盘 → 置信度重校准
    │
    ▼
最终输出: 单式5注 + 12种复式方案
```

### 1.2 核心变化（对比 V1.0）

| 维度 | V1.0 | V2.1.0 |
|------|------|--------|
| 池数 | 5池（热/冷/均衡/趋势/质数） | 6池（热/冷/均衡/趋势/博弈/遗传）+ 第7模式池 |
| 策略融合引擎 | 5组×5维 | 5组×5维（参数同步更新） |
| 重号策略 | 静态 | 动态（和值>120无重号预期, <70少重号） |
| 尾号处理 | 无 | 尾号聚合检测 |
| AC值 | 仅作为统计指标 | AC值跟踪评分（基于近100期概率分布） |
| 偏差校准 | 无 | 偏差仪表盘 + 置信度重校准 |
| 奇偶约束 | 强制3:2或2:3 | 放宽，允许全偶/全奇路径 |
| 数据同步 | 单源 | 双源自动切换 |

## 二、StrategyFusionEngine 设计（本文件核心）

### 2.1 设计定位

`StrategyFusionEngine` 是 V2.1.0 管线中的一路独立策略线，与六池采样、模式识别并列。它不取代多池采样，而是提供一种**多维度权重融合**的差异化思路。

### 2.2 核心设计思想

```
新架构思路（与旧五池对比）：
  旧：组1 = 热号排序 top-5（单一过滤器）
  新：组1 = 热号(30%) + 重号(25%) + 奇偶均衡(20%) + 小三区(15%) + 遗漏(10%)

核心变化：每组不再是一个"号码过滤器"，而是多维度加权融合
```

### 3.2 组1：热号重号均衡策略

**设计思路**：以历史热号为驱动，辅助重号预测，平衡奇偶和大小区间。

```
维度权重：
  热号(hot)      : 30%  → 频率最高的号码
  重号(repeat)   : 25%  → 上期出现过的号码（高概率）
  奇偶均衡       : 20%  → 强制奇偶比3:2或2:3
  小三区         : 15%  → 优先前区小号(≤18)
  遗漏值(missing): 10%  → 校准，避免完全偏向热号

硬约束：
  - 和值范围：80-110
  - 奇偶比：3:2, 2:3, 4:1, 1:4（V2.1.0放宽）
  - 跨度：20-32
  - AC值 ≥ 6
  - 三区必须有号码
  - 连号组数：1-2组

软约束（优化目标）：
  - 和值偏好：88-102
  - 跨度偏好：24-28
  - 三区偏好：221（2:2:1）
  - AC值偏好：8
  - 共现增强：开启

融合算法：加权投票（Weighted Voting）
  - 最简单直接的融合方式
  - 适合作为baseline策略
  - 每号码计算：score = Σ(维度得分 × 权重)
```

### 3.3 组2：冷号遗漏反转策略

**设计思路**：基于"彩票均值回归"理论，冷号和遗漏值大的号码迟早回补。

```
维度权重：
  遗漏值(missing): 35%  → Beta-Bayes后验，冷号优先
  冷号(cold)     : 25%  → 频率最低的号码
  趋势反弹(trend): 15%  → 近期频率上升（均值回归信号）
  质数(prime)    : 15%  → 质数有一定的出现规律
  共现矩阵(cooc) : 10%  → 历史共现号码增强

硬约束：
  - 和值范围：75-115
  - 奇偶比：3:2, 2:3, 4:1（V2.1.0新增4:1）
  - 跨度：18-34（允许较大跨度）
  - AC值 ≥ 5（宽松）
  - 三区必须有号码
  - 连号组数：0-2组

软约束：
  - 和值偏好：80-100
  - 跨度偏好：22-30
  - 三区偏好：212
  - AC值偏好：7
  - 共现增强：开启

融合算法：贝叶斯融合（Bayesian Fusion）
  - 后验 = 先验^ml_weight × 似然^(1-ml_weight)
  - 先验来自维度评分，似然来自历史频率
  - 加入遗漏值Beta-Binomial调整
  - 比加权投票更能量化不确定性
```

### 3.4 组3：趋势动量策略

**设计思路**：基于动量效应，近期表现强势的号码继续强势。

```
维度权重：
  趋势动量(trend) : 35%  → 近期5期vs15期频率差（核心因子）
  重号(repeat)   : 20%  → 延续上期号码
  临号共振(adj)  : 20%  → 与上期号码相邻（边界效应）
  热号(hot)      : 15%  → 频率基础
  遗漏值(missing): 10%  → 校准

硬约束：
  - 和值范围：85-115
  - 奇偶比：3:2, 2:3, 4:1, 1:4（允许极端）
  - 跨度：20-30
  - AC值 ≥ 6
  - 三区必须有号码
  - 连号组数：0-3组（允许较多连号）

软约束：
  - 和值偏好：90-108
  - 三区偏好：122（1:2:2，偏向中间区域）
  - AC值偏好：8
  - 共现增强：开启

融合算法：博弈论均衡（Game Theory Equilibrium）
  - 考虑大众偏好回避
  - 热门号段惩罚（1-10, 10, 20, 30等整十）
  - 高号区奖励（冷门区域）
  - 遗漏极端奖励
```

### 3.5 组4：博弈论冷门策略

**设计思路**：在博弈论框架下，选择最不被大众选择的号码。

```
维度权重：
  冷号(cold)      : 30%  → 频率最低（反向思维）
  质数(prime)     : 25%  → 质数有独特规律
  高号区间        : 20%  → 优先28以上的号码（冷门区域）
  遗漏值(missing) : 15%  → 大遗漏优先
  共现规避(cooc)  : 10%  → 避免高频共现组合

硬约束：
  - 和值范围：90-125（偏高和值）
  - 奇偶比：全部允许（5:0, 0:5等极端情况，V2.1.0新增）
  - 跨度：25-34（大跨度）
  - AC值 ≥ 5
  - 三区必须有号码（允许空区）
  - 连号组数：0-1组（避免连号）

软约束：
  - 和值偏好：95-115（高和值）
  - 奇偶偏好：4:1（偏奇）
  - 跨度偏好：28-32
  - AC值偏好：9
  - 共现增强：**关闭**（规避而非增强）

融合算法：博弈论均衡（与组3相同，但参数不同）
  - 冷门驱动：低号段惩罚、高号段奖励
  - 生日号规避：1-12区间惩罚
  - 质数优先：质数天然稀缺，策略加分
  - ML权重：0.25（较高，依赖模型预测）
```

### 3.6 组5：ML概率融合策略

**设计思路**：最均衡的五维融合，充分利用ML模型的概率输出。

```
维度权重：
  热号(hot)    : 20%  → 基础频率
  遗漏值       : 20%  → Beta-Bayes后验
  趋势动量    : 20%  → 动量效应
  重号(repeat): 20%  → 延续性
  共现矩阵    : 20%  → 组合优化
  （五维完全均等，无单一主导因子）

硬约束：
  - 和值范围：75-120
  - 奇偶比：3:2, 2:3, 4:1, 1:4
  - 跨度：18-32
  - AC值 ≥ 6
  - 三区必须有号码
  - 连号组数：0-2组

软约束：
  - 和值偏好：85-105（居中）
  - 三区偏好：221（均衡分布）
  - AC值偏好：8
  - 共现增强：开启

融合算法：ML概率融合（ML Probability Fusion）
  - ML权重：0.4（最高）
  - 结合ML模型的概率输出和规则维度评分
  - 可对接现有的DLTPredictorUpgraded的64维特征ML模型
  - 动态校准：近期命中率加成
```

## 五、融合算法选型评估

### 5.1 五种融合算法对比

| 算法 | 复杂度 | 不确定性量化 | 计算速度 | 适用场景 | 推荐组 |
|------|--------|-------------|---------|---------|-------|
| **加权投票** | ⭐ | ❌ | ⚡⚡⚡ | 快速baseline，组1 | ✓✓✓ |
| **贝叶斯融合** | ⭐⭐ | ✅ | ⚡⚡ | 小样本、遗漏反转 | ✓✓ |
| **ML概率融合** | ⭐⭐⭐ | ✅ | ⚡⚡ | 有ML模型的场景，组5 | ✓✓✓ |
| **遗传算法优化** | ⭐⭐⭐⭐ | ❌ | ⚡ | 全局优化、最终精选 | ✓ |
| **博弈论均衡** | ⭐⭐⭐ | ⚡ | ⚡⚡ | 考虑对手行为，组3/4 | ✓✓✓ |

### 5.2 各算法详细说明

**① 加权投票法（Weighted Voting）**
```python
score(num) = Σ (维度得分_i × 权重_i)
```
- 优点：简单、快速、易解释
- 缺点：无法量化不确定性，无法自适应
- 适用：热号驱动策略（组1）

**② 贝叶斯融合（Bayesian Fusion）**
```python
后验 = 先验^α × 似然^(1-α)
先验 = 维度加权得分
似然 = Beta-Binomial后验（基于历史频率）
α = ml_model_weight
```
- 优点：不确定性量化好，适合小样本
- 缺点：需要定义先验，计算稍复杂
- 适用：冷号反转策略（组2）

**③ ML概率融合（ML Probability Fusion）**
```python
score = 0.4 × ML_prob + 0.6 × Rule_score
ML_prob = LR/GB模型输出的概率
Rule_score = 维度加权得分
```
- 优点：结合ML学习和规则约束
- 缺点：依赖ML模型质量
- 适用：已有ML模型的场景（组5）

**④ 遗传算法优化（Genetic Optimization）**
```python
适应度函数 = base_score × diversity_boost
选择 → 交叉 → 变异 → 迭代
```
- 优点：全局搜索能力强
- 缺点：收敛慢，需调参
- 适用：最终精选阶段

**⑤ 博弈论均衡（Game Theory Equilibrium）**
```python
最终分数 = 基础分 + 博弈调整
博弈调整 = 大众回避奖励 + 冷门区域奖励 + 遗漏极端奖励
```
- 优点：考虑竞争对手行为
- 缺点：博弈参数需校准
- 适用：趋势策略和冷门策略（组3、组4）

```python
        """完整流程入口"""
```

### 6.3 关键评分公式

```
候选综合得分 = 基础分 × (0.6 + 0.4 × 软约束得分)

其中：
  基础分 = Σ(维度得分_i × 权重_i)  [for i in dimensions]

最终排序分 = 候选分 × (0.7 + 0.3 × 多样性得分) × (0.9 + 0.1 × 近期加成)

其中：
  多样性得分 = max(0.5, 1.0 - 平均重叠度)
  近期加成 = Σ(近期N期重叠数) / (N × 2)
```

### 7.2 三路合成流程

```
dlt_fusion_complete.py predict():
  1. 六池采样 → top-40 候选
  2. 模式识别 → 第7池候选 + 评分增强
  3. SFE → 5组候选（5组×30候选 = 150候选）
  4. 三路合并 → 去重
  5. 约束引擎二次过滤
  6. SkipRepeatBooster / SmartRepeatPenalty 打分调整

  8. 遗传算法最终精选 → top-5单式 + 12种复式
```

## 八、V2.1.0 新增后处理模块

> 这些模块在 SFE 输出之后运行，作用于最终候选集

### 8.1 尾号聚合检测（Tail Aggregation）

- 计算候选组合的尾号（个位数字）种类数
- 尾号 ≤ 3 种 → 降分 3%（过于集中）
- 恰好 4 种（1组重复）→ 加 1%（典型分布）

### 8.2 AC值跟踪评分（AC Value Tracking）

- 基于最近100期AC值的概率分布
- 对每个候选组合计算AC值，对比历史概率
- 高概率AC值 → 加分；低概率AC值 → 扣分

### 8.3 偏差仪表盘（Deviation Dashboard）

- 实时监控三个维度偏离程度：
  - 和值偏差（近10期均值 vs 总体均值）
  - 跨度偏差
  - 大小号比例偏差
- 偏差过大时，补偿生成被低估方向的候选

### 8.4 置信度重校准（Confidence Recalibration）

- 根据偏差修正最终评分
- 偏差越大 → 置信度降低 → 重校准因子动态调整

### 9.1 预期效果

| 指标 | 旧架构 | 新架构 | 改善 |

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
