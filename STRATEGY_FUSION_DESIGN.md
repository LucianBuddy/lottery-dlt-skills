## 一、现有FivePoolSampler架构问题分析

### 1.1 当前架构回顾

```python
pools = ['hot', 'cold', 'balance', 'trend', 'prime']
```

每组方案对应单一池子：组1=热号池、组2=冷号池、组3=均衡池、组4=趋势池、组5=质数池。

### 1.2 五大核心问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **单一维度陷阱** | 每组仅依赖一个维度 | 预测质量高度依赖单一特征有效性 |
| **策略同质化** | 5组方案本质是同一策略的变体 | 无法覆盖多种投注场景 |
| **无约束机制** | 无硬/软约束概念 | 组合可能违背基本统计规律 |
| **无融合算法** | 各池独立工作，无交叉融合 | 浪费13维特征的丰富信息 |
| **多样性缺失** | 组间可能高度重叠 | 推荐结果缺乏差异性 |

### 1.3 旧架构的根本缺陷

```
旧架构思路：
  组1 = 热号排序 top-5
  组2 = 冷号排序 top-5
  组3 = 均衡池混合
  组4 = 趋势上升号
  组5 = 质数优先

问题：每组只是一个"号码过滤器"，不是真正的"策略"
```

## 二、新架构设计：多维度策略融合引擎

### 2.1 核心设计思想

```
新架构思路：
  组1 = 热号(30%) + 重号(25%) + 奇偶均衡(20%) + 小三区(15%) + 遗漏(10%)
  组2 = 遗漏(35%) + 冷号(25%) + 趋势反弹(15%) + 质数(15%) + 共现(10%)
  组3 = 趋势(35%) + 重号(20%) + 临号共振(20%) + 热号(15%) + 遗漏(10%)
  组4 = 冷号(30%) + 质数(25%) + 高号区间(20%) + 遗漏(15%) + 共现规避(10%)
  组5 = 均衡五维(各20%)

核心变化：每组不再是单一过滤器，而是多维度加权融合
```

### 3.2 组1：热号重号均衡策略

**设计思路**：以历史热号为驱动，辅助重号预测，平衡奇偶和大小区间。

```
维度权重：
  热号(hot)      : 30%  → 频率最高的号码
  重号(repeat)   : 25%  → 上期出现过的号码（高概率）
  奇偶均衡       : 20%  → 强制奇偶比3:2或2:3
  小三区         : 15%  → 优先前区小号(≤18)
  遗漏值(missing): 10%  → 校准，避免完全偏向热号

硬约束：
  - 和值范围：80-110
  - 奇偶比：3:2 或 2:3
  - 跨度：20-32
  - AC值 ≥ 6
  - 三区必须有号码（允许空区惩罚）
  - 连号组数：1-2组

软约束（优化目标）：
  - 和值偏好：88-102
  - 跨度偏好：24-28
  - 三区偏好：221（2:2:1）
  - AC值偏好：8
  - 共现增强：开启

融合算法：加权投票（Weighted Voting）
  - 最简单直接的融合方式
  - 适合作为baseline策略
  - 每号码计算：score = Σ(维度得分 × 权重)
```

### 3.3 组2：冷号遗漏反转策略

**设计思路**：基于"彩票均值回归"理论，冷号和遗漏值大的号码迟早回补。

```
维度权重：
  遗漏值(missing): 35%  → Beta-Bayes后验，冷号优先
  冷号(cold)     : 25%  → 频率最低的号码
  趋势反弹(trend): 15%  → 近期频率上升（均值回归信号）
  质数(prime)    : 15%  → 质数有一定的出现规律
  共现矩阵(cooc) : 10%  → 历史共现号码增强

硬约束：
  - 和值范围：75-115
  - 奇偶比：3:2, 2:3, 4:1（允许偏奇）
  - 跨度：18-34（允许较大跨度）
  - AC值 ≥ 5（宽松）
  - 三区必须有号码
  - 连号组数：0-2组

软约束：
  - 和值偏好：80-100
  - 跨度偏好：22-30
  - 三区偏好：212
  - AC值偏好：7
  - 共现增强：开启

融合算法：贝叶斯融合（Bayesian Fusion）
  - 后验 = 先验^ml_weight × 似然^(1-ml_weight)
  - 先验来自维度评分，似然来自历史频率
  - 加入遗漏值Beta-Binomial调整
  - 比加权投票更能量化不确定性
```

### 3.4 组3：趋势动量策略

**设计思路**：基于动量效应，近期表现强势的号码继续强势。

```
维度权重：
  趋势动量(trend) : 35%  → 近期5期vs15期频率差（核心因子）
  重号(repeat)   : 20%  → 延续上期号码
  临号共振(adj)  : 20%  → 与上期号码相邻（边界效应）
  热号(hot)      : 15%  → 频率基础
  遗漏值(missing): 10%  → 校准

硬约束：
  - 和值范围：85-115
  - 奇偶比：3:2, 2:3, 4:1, 1:4（允许极端）
  - 跨度：20-30
  - AC值 ≥ 6
  - 三区必须有号码
  - 连号组数：0-3组（允许较多连号）

软约束：
  - 和值偏好：90-108
  - 三区偏好：122（1:2:2，偏向中间区域）
  - AC值偏好：8
  - 共现增强：开启

融合算法：博弈论均衡（Game Theory Equilibrium）
  - 考虑大众偏好回避
  - 热门号段惩罚（1-10, 10, 20, 30等整十）
  - 高号区奖励（冷门区域）
  - 遗漏极端奖励
```

### 3.5 组4：博弈论冷门策略

**设计思路**：在博弈论框架下，选择最不被大众选择的号码。

```
维度权重：
  冷号(cold)      : 30%  → 频率最低（反向思维）
  质数(prime)     : 25%  → 质数有独特规律
  高号区间        : 20%  → 优先28以上的号码（冷门区域）
  遗漏值(missing) : 15%  → 大遗漏优先
  共现规避(cooc)  : 10%  → 避免高频共现组合

硬约束：
  - 和值范围：90-125（偏高和值）
  - 奇偶比：全部允许（5:0, 0:5等极端情况）
  - 跨度：25-34（大跨度）
  - AC值 ≥ 5
  - 三区必须有号码（允许空区）← 注意：zone_required=False
  - 连号组数：0-1组（避免连号）

软约束：
  - 和值偏好：95-115（高和值）
  - 奇偶偏好：4:1（偏奇）
  - 跨度偏好：28-32
  - AC值偏好：9
  - 共现增强：**关闭**（规避而非增强）

融合算法：博弈论均衡（与组3相同，但参数不同）
  - 冷门驱动：低号段惩罚、高号段奖励
  - 生日号规避：1-12区间惩罚
  - 质数优先：质数天然稀缺，策略加分
  - ML权重：0.25（较高，依赖模型预测）
```

### 3.6 组5：ML概率融合策略

**设计思路**：最均衡的五维融合，充分利用ML模型的概率输出。

```
维度权重：
  热号(hot)    : 20%  → 基础频率
  遗漏值       : 20%  → Beta-Bayes后验
  趋势动量    : 20%  → 动量效应
  重号(repeat): 20%  → 延续性
  共现矩阵    : 20%  → 组合优化
  （五维完全均等，无单一主导因子）

硬约束：
  - 和值范围：75-120
  - 奇偶比：3:2, 2:3, 4:1, 1:4
  - 跨度：18-32
  - AC值 ≥ 6
  - 三区必须有号码
  - 连号组数：0-2组

软约束：
  - 和值偏好：85-105（居中）
  - 三区偏好：221（均衡分布）
  - AC值偏好：8
  - 共现增强：开启

融合算法：ML概率融合（ML Probability Fusion）
  - ML权重：0.4（最高）
  - 结合ML模型的概率输出和规则维度评分
  - 可对接现有的DLTPredictorUpgraded的64维特征ML模型
  - 动态校准：近期命中率加成
```

## 五、融合算法选型评估

### 5.1 五种融合算法对比

| 算法 | 复杂度 | 不确定性量化 | 计算速度 | 适用场景 | 推荐组 |
|------|--------|-------------|---------|---------|-------|
| **加权投票** | ⭐ | ❌ | ⚡⚡⚡ | 快速baseline，组1 | ✓✓✓ |
| **贝叶斯融合** | ⭐⭐ | ✅ | ⚡⚡ | 小样本、遗漏反转 | ✓✓ |
| **ML概率融合** | ⭐⭐⭐ | ✅ | ⚡⚡ | 有ML模型的场景，组5 | ✓✓✓ |
| **遗传算法优化** | ⭐⭐⭐⭐ | ❌ | ⚡ | 全局优化、最终精选 | ✓ |
| **博弈论均衡** | ⭐⭐⭐ | ⚡ | ⚡⚡ | 考虑对手行为，组3/4 | ✓✓✓ |

### 5.2 各算法详细说明

**① 加权投票法（Weighted Voting）**
```python
score(num) = Σ (维度得分_i × 权重_i)
```
- 优点：简单、快速、易解释
- 缺点：无法量化不确定性，无法自适应
- 适用：热号驱动策略（组1）

**② 贝叶斯融合（Bayesian Fusion）**
```python
后验 = 先验^α × 似然^(1-α)
先验 = 维度加权得分
似然 = Beta-Binomial后验（基于历史频率）
α = ml_model_weight
```
- 优点：不确定性量化好，适合小样本
- 缺点：需要定义先验，计算稍复杂
- 适用：冷号反转策略（组2）

**③ ML概率融合（ML Probability Fusion）**
```python
score = 0.4 × ML_prob + 0.6 × Rule_score
ML_prob = RandomForest/GB模型输出的概率
Rule_score = 维度加权得分
```
- 优点：结合ML学习和规则约束
- 缺点：依赖ML模型质量
- 适用：已有ML模型的场景（组5）

**④ 遗传算法优化（Genetic Optimization）**
```python
适应度函数 = base_score × diversity_boost
选择 → 交叉 → 变异 → 迭代
```
- 优点：全局搜索能力强
- 缺点：收敛慢，需调参
- 适用：最终精选阶段

**⑤ 博弈论均衡（Game Theory Equilibrium）**
```python
最终分数 = 基础分 + 博弈调整
博弈调整 = 大众回避奖励 + 冷门区域奖励 + 遗漏极端奖励
```
- 优点：考虑竞争对手行为
- 缺点：博弈参数需校准
- 适用：趋势策略和冷门策略（组3、组4）

```python
class StrategyFusionEngine:
    def __init__(self, draws, n_groups=5):
        """初始化：加载历史数据，构建所有组件"""

    def generate_group_1(self) -> List[ScoredCandidate]:
        """热号重号均衡策略"""

        """冷号遗漏反转策略"""

        """趋势动量策略"""

    def generate_group_4(self) -> List[ScoredCandidate]:
        """博弈论冷门策略"""

    def generate_group_5(self) -> List[ScoredCandidate]:
        """ML概率融合策略"""

    def generate_all_groups(self, n_per_group=5) -> Dict[int, List[ScoredCandidate]]:
        """生成所有组候选"""

    def select_and_rank(self, all_candidates, top_n=5) -> List[ScoredCandidate]:
        """全局去重 + 多样性boost + 近期加成 + 排序"""

    def run(self, n_per_group=5, top_n=5) -> List[ScoredCandidate]:
        """完整流程入口"""
```

### 6.3 关键评分公式

```
候选综合得分 = 基础分 × (0.6 + 0.4 × 软约束得分)

其中：
  基础分 = Σ(维度得分_i × 权重_i)  [for i in dimensions]

最终排序分 = 候选分 × (0.7 + 0.3 × 多样性得分) × (0.9 + 0.1 × 近期加成)

其中：
  多样性得分 = max(0.5, 1.0 - 平均重叠度)
  近期加成 = Σ(近期N期重叠数) / (N × 2)
```

### 8.1 预期效果

## 十、总结

### 核心创新

1. **从单维度到多维度**：每组不再是单一池子，而是5个维度的加权融合
2. **约束层次化**：硬约束保证基本规律，软约束引导优化方向
3. **多算法融合**：加权投票+贝叶斯+博弈论，覆盖不同预测逻辑
4. **多样性保证**：全局去重+多样性boost，避免推荐结果千篇一律
5. **可扩展架构**：策略可热插拔，维度可动态调整

### 与旧架构的关键区别

| 维度 | 旧架构 | 新架构 |
|------|--------|--------|
| 每组策略 | 单一池子 | 多维度融合 |
| 约束处理 | 无 | 硬约束+软约束 |
| 融合算法 | 无 | 5种可选 |
| 多样性 | 无保证 | 多样性boost |
| 输出 | 按game_score排序 | 多维度综合评分 |
